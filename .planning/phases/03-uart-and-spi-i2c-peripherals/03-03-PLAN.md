- Assert: at least one event with `type === "uart_tx"` was received

Test 2: "UART RX echo round-trip via WebSocket" (UART-03)
- Compile and run uart-hello sample (speed=10 for reasonable timing)
- Connect WebSocket
- Wait for the initial "Hello from STM32!" uart_tx event
- Send a `{ type: "uart_rx", data: "A" }` message via WebSocket
- Wait for an echo uart_tx event containing "A"
- Assert: received a uart_tx event with data containing "A" (the echoed character)
- Timeout: 10 seconds for the round-trip
- Clean up: close WS, stop simulation

Test 3: "uart_rx with invalid data is ignored" (robustness)
- Compile and run uart-hello sample
- Connect WebSocket
- Send `{ type: "uart_rx", data: "" }` (empty string) -- should not crash
- Send `{ type: "uart_rx" }` (missing data) -- should not crash
- Wait 1 second
- Assert: simulation is still running (no crash), clean up

Each test has 15-second timeout for CI reliability.
