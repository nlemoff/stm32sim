<objective>
Wire the existing no-op UART, SPI, and I2C HAL stubs into the simulator event pipeline and add server-side UART input forwarding, completing the backend for all Phase 3 peripherals.

Purpose: The C HAL stubs currently return HAL_OK but produce no events. This plan transforms them into functional simulation stubs that emit JSON events (uart_tx, spi_transfer, i2c_transfer) following the exact pattern established by hal_gpio.c. It also enables bidirectional UART by extending the stdin input channel (established in Phase 2 for gpio_input) to handle uart_rx messages. Sample firmware programs provide verification artifacts.

Output: Functional HAL stubs for UART/SPI/I2C, server-side uart_rx forwarding, and two sample firmware programs (uart-hello, spi-loopback).
</objective>

3. Implement `sim_uart_rx_pop(out, max_bytes)`: Pop up to max_bytes from the ring buffer into out. Return number of bytes actually popped.

**process-manager.ts** -- Add `sendUartInput` function following the `sendGpioInput` pattern:

```typescript
export function sendUartInput(simId: string, data: string): void {
  const sim = simulationStore.get(simId);
  if (!sim) return;
  try {
    const json = JSON.stringify({ type: "uart_rx", data }) + "\n";
    const stdin = sim.process.stdin as any;
    if (stdin) {
      stdin.write(json);
      stdin.flush();
    }
  } catch {
    // Process may have already exited
  }
}
```
