  <action>
    **Create `src/server/compiler/errors.ts` first (pure function, no deps):**

    Export `DiagnosticMessage` interface: `{ file: string, line: number, column: number, severity: "error" | "warning" | "note", message: string, option?: string }`.

    Export `parseGccDiagnostics(rawStderr: string): { errors: DiagnosticMessage[], warnings: DiagnosticMessage[] }`.
    Implementation:
    - Try `JSON.parse(rawStderr)` -- GCC 13 outputs a JSON array to stderr with `-fdiagnostics-format=json`
    - If parse succeeds, map each diagnostic: extract `kind` as severity, `message`, `locations[0].caret.file/line/column`, `option`
    - Filter into errors (kind === "error") and warnings (kind === "warning" or "note")
    - If JSON.parse fails, this is likely a linker error or mixed output. Try to extract JSON portion (find first `[` and last `]`). If that also fails, return the entire raw string as a single error with file "linker", line 0, column 0, severity "error".

    **Create `src/server/compiler/compile.ts`:**

    Export `CompileResult` interface: `{ success: boolean, compilationId?: string, binaryPath?: string, errors: DiagnosticMessage[], warnings: DiagnosticMessage[], rawOutput: string }`.

    Export `async function compile(code: string): Promise<CompileResult>`.
    Implementation follows RESEARCH.md Pattern 5:
    - Create temp directory with `mkdtemp(join(tmpdir(), "stm32sim-"))`
    - Write user code to `{tmpDir}/main.c`
    - Build gcc args: `["gcc", "-fdiagnostics-format=json", "-I", HAL_INCLUDE_PATH, "-o", outputBinary, userFile, ...HAL_SOURCES, "-lm"]`
    - HAL_INCLUDE_PATH = resolve from `import.meta.dir` to `../../hal/include` (or use `join(process.cwd(), "hal/include")` -- prefer process.cwd for reliability)
    - HAL_SOURCES = glob all `.c` files in `hal/src/`
    - Run with `Bun.spawnSync()`, capture stderr
    - Parse stderr with `parseGccDiagnostics()`
    - On success: return compilationId (crypto.randomUUID()), binaryPath, empty errors, parsed warnings
    - On failure: return parsed errors and warnings, no binaryPath

    Also export `async function cleanupCompilation(compilationId: string, binaryPath: string): Promise<void>` that removes the temp directory using `rm(dirname(binaryPath), { recursive: true, force: true })`.

    **Write tests following TDD: RED first, then GREEN.**

    - Test 3: Events include sim_start and gpio_write types with correct structure
    - Test 4: stopSimulation() kills the subprocess (process exits)
    - Test 5: Simulation auto-terminates after configurable timeout (test with 2-second timeout)
    - Test 6: getSimulation() returns simulation state while running, undefined after stopped
    - Test 7: onExit callback fires when process exits naturally or is killed

  <action>
    **Create `src/server/runner/stdout-parser.ts` first:**

    Export `SimulationEvent` interface: `{ type: string, timestamp_ms: number, data: Record<string, unknown> }`.

    Export `function parseEventLine(line: string): SimulationEvent | null` -- parses a single JSON line, returns null on parse failure.

    Export `async function* streamEvents(reader: ReadableStreamDefaultReader<Uint8Array>): AsyncGenerator<SimulationEvent>` -- reads chunks, splits on newlines (buffering partial lines), yields parsed SimulationEvent objects. Skip lines that don't parse as valid JSON.

    **Create `src/server/state/simulation.ts`:**

    Export `SimulationState` interface: `{ id: string, process: Subprocess, binaryPath: string, compilationId: string, timeout: Timer | null, startedAt: number, speed: number }`.

    Export `simulationStore` as a `Map<string, SimulationState>`.

    **Create `src/server/runner/process-manager.ts`:**

    Export `interface StartOptions { compilationId: string, binaryPath: string, speed?: number, timeoutMs?: number, onEvent?: (event: SimulationEvent) => void, onExit?: (exitCode: number | null, signal: string | null) => void }`.

    Export `async function startSimulation(options: StartOptions): Promise<string>`:
    - Generate simulation ID with `crypto.randomUUID()`
    - Spawn binary with `Bun.spawn([binaryPath], { stdout: "pipe", stderr: "pipe", env: { ...process.env, SIM_SPEED: String(speed || 1.0) } })`
    - Set timeout with `setTimeout(() => stopSimulation(id), timeoutMs || 30000)`
    - Store in simulationStore
    - Start async stdout reading loop using `streamEvents()` from stdout-parser, calling `onEvent` for each event
    - On process exit (via `proc.exited.then()`): call `onExit`, clear timeout, remove from store

    Export `function stopSimulation(id: string): boolean`:
    - Get simulation from store, return false if not found
    - Kill process with `proc.kill()`
    - Clear timeout
    - Remove from store
    - Return true

    Export `function getSimulation(id: string): SimulationState | undefined`.

    Export `function stopAllSimulations(): void` -- iterate store, kill all. Used for server shutdown.

    **Write tests using compiled blink sample from Plan 01.**
    Tests should compile the blink sample first (using compile function from Task 1), then use the resulting binary to test the runner. Set short timeouts (2-3 seconds) for tests. Use `afterAll` to clean up any running processes.
