  <action>
    **Step 1: Install Bun and initialize project.**
    Run `curl -fsSL https://bun.sh/install | bash` to install Bun. Then initialize the project:
    - Create `package.json` with name "stm32sim", type "module", scripts: { "dev": "bun run src/server/index.ts", "test": "bun test" }
    - Create `bunfig.toml` with `[test]` section setting `preload` and test root to `tests/`
    - Create `tsconfig.json` with strict mode, ES2022 target, module "ESNext", moduleResolution "bundler"
    - Run `bun add -d @types/bun` for TypeScript types

    **Step 2: Create mock HAL header files.**
    Follow the RESEARCH.md architecture and code examples precisely. Create headers that match real STM32F4 HAL signatures so user code looks like authentic STM32CubeIDE output.

    Headers to create:
    - `hal/include/stm32f4xx.h` -- Top-level device header. Defines `__STM32F4xx_HAL_H` include guard, includes `stdint.h`, provides base type forward declarations.
    - `hal/include/stm32f4xx_hal.h` -- HAL umbrella header. Includes hal_conf, hal_def, hal_gpio, hal_rcc, hal_cortex, hal_uart, hal_spi, hal_i2c. Declares `HAL_Init()`, `HAL_DeInit()`, `HAL_GetTick()`, `void HAL_Delay(uint32_t)`, `void SystemClock_Config(void)`.
    - `hal/include/stm32f4xx_hal_conf.h` -- Module enable macros: `HAL_MODULE_ENABLED`, `HAL_GPIO_MODULE_ENABLED`, `HAL_UART_MODULE_ENABLED`, `HAL_SPI_MODULE_ENABLED`, `HAL_I2C_MODULE_ENABLED`, `HAL_RCC_MODULE_ENABLED`, `HAL_CORTEX_MODULE_ENABLED`.
    - `hal/include/stm32f4xx_hal_def.h` -- Exactly as shown in RESEARCH.md: `HAL_StatusTypeDef` enum, `__IO`/`__I` macros, `FlagStatus`, `FunctionalState`, `UNUSED` macro.
    - `hal/include/stm32f4xx_hal_gpio.h` -- Exactly as shown in RESEARCH.md: `GPIO_TypeDef` struct with register fields, `GPIO_InitTypeDef`, `GPIO_PinState` enum, all GPIO_PIN_0 through GPIO_PIN_15 and GPIO_PIN_All defines, mode defines (INPUT, OUTPUT_PP, OUTPUT_OD, AF_PP, AF_OD, ANALOG), pull defines, speed defines, GPIOA-GPIOE extern instances and macros, function prototypes for Init/DeInit/ReadPin/WritePin/TogglePin/LockPin.
    - `hal/include/stm32f4xx_hal_rcc.h` -- RCC clock enable macros as no-op stubs: `__HAL_RCC_GPIOA_CLK_ENABLE()` through `__HAL_RCC_GPIOE_CLK_ENABLE()` defined as `((void)0)`. Also `__HAL_RCC_USART1_CLK_ENABLE()`, `__HAL_RCC_USART2_CLK_ENABLE()`, `__HAL_RCC_SPI1_CLK_ENABLE()`, `__HAL_RCC_I2C1_CLK_ENABLE()`.
    - `hal/include/stm32f4xx_hal_cortex.h` -- NVIC stubs: `HAL_NVIC_SetPriority()`, `HAL_NVIC_EnableIRQ()`, `HAL_NVIC_DisableIRQ()` as function prototypes (implementations are no-ops in hal_system.c).
    - `hal/include/stm32f4xx_hal_uart.h` -- UART types: `UART_HandleTypeDef` with Instance pointer, Init struct (BaudRate, WordLength, StopBits, Parity, Mode, HwFlowCtl, OverSampling), function prototypes for `HAL_UART_Init()`, `HAL_UART_Transmit()`, `HAL_UART_Receive()`. All return `HAL_StatusTypeDef`. Define `USART_TypeDef` struct and `USART1`/`USART2` extern instances.
    - `hal/include/stm32f4xx_hal_spi.h` -- SPI types: `SPI_HandleTypeDef` with Init struct, function prototypes for `HAL_SPI_Init()`, `HAL_SPI_Transmit()`, `HAL_SPI_Receive()`, `HAL_SPI_TransmitReceive()`. Define `SPI_TypeDef` struct and `SPI1` extern instance.
    - `hal/include/stm32f4xx_hal_i2c.h` -- I2C types: `I2C_HandleTypeDef` with Init struct, function prototypes for `HAL_I2C_Init()`, `HAL_I2C_Master_Transmit()`, `HAL_I2C_Master_Receive()`. Define `I2C_TypeDef` struct and `I2C1` extern instance.

    **Step 3: Create mock HAL implementation files.**

    - `hal/src/sim_runtime.h` -- Declares `void sim_init(void)`, `void sim_emit_event(const char *type, const char *data_fmt, ...)`, `void sim_cleanup(void)`. Also declares `extern double sim_speed_multiplier` for use by HAL_Delay.
    - `hal/src/sim_runtime.c` -- Exactly as shown in RESEARCH.md: `sim_init()` disables stdout buffering with `setvbuf(stdout, NULL, _IONBF, 0)`, reads `SIM_SPEED` env var, emits `sim_start` event. `sim_emit_event()` uses `clock_gettime(CLOCK_MONOTONIC)` for timestamps, formats line-delimited JSON to stdout. Store a `start_time` at init and compute relative timestamps from it.
    - `hal/src/sim_main.c` -- Provides the `main()` entry point that calls `sim_init()`, then calls the user's `int app_main(void)`. IMPORTANT: The user's code defines `main()` normally in their samples. Use a linker trick: `sim_main.c` provides `__attribute__((constructor))` function that calls `sim_init()` before main runs, and `__attribute__((destructor))` that calls `sim_cleanup()`. This way user code can have a normal `main()` function.
    - `hal/src/hal_gpio.c` -- GPIO stub implementations following RESEARCH.md pattern exactly. In-memory state arrays for 5 ports (GPIOA-GPIOE) x 16 pins. Allocate `GPIO_TypeDef _GPIOA_Instance` through `_GPIOE_Instance` as global structs. `HAL_GPIO_Init()` records pin modes and emits `gpio_init` event. `HAL_GPIO_WritePin()` updates ODR and emits `gpio_write` with port letter (A-E), individual pin number (0-15, decoded from bitmask), and state (0/1). `HAL_GPIO_TogglePin()` XORs ODR and emits `gpio_write`. `HAL_GPIO_ReadPin()` reads from IDR. CRITICAL: Decode bitmask to individual pin numbers -- if multiple pins are in the mask, emit one event per pin. Use port letters ("A", "B", etc.) not indices in the JSON for readability.
    - `hal/src/hal_system.c` -- `HAL_Init()` calls `sim_init()` and returns `HAL_OK`. `HAL_DeInit()` returns `HAL_OK`. `HAL_GetTick()` returns milliseconds since init using `clock_gettime`. `HAL_Delay(uint32_t Delay)` emits delay event, then `usleep((uint32_t)(Delay / sim_speed_multiplier) * 1000)`. `SystemClock_Config()` is a no-op. NVIC functions from cortex.h are no-ops returning void or `HAL_OK`.
    - `hal/src/hal_uart.c` -- All functions are no-op stubs returning `HAL_OK`. Include a comment: "// UART behavior wired in Phase 3. These stubs allow UART code to compile."  Allocate `USART_TypeDef _USART1_Instance, _USART2_Instance` as globals.
    - `hal/src/hal_spi.c` -- Same pattern as UART: no-op stubs returning `HAL_OK`, with comment about Phase 3. Allocate `SPI_TypeDef _SPI1_Instance`.
    - `hal/src/hal_i2c.c` -- Same pattern: no-op stubs returning `HAL_OK`. Allocate `I2C_TypeDef _I2C1_Instance`.

    **Step 4: Create three sample firmware programs.**

    All samples must include educational comments and look like authentic STM32CubeIDE output per user decision.

    - `samples/blink/main.c` -- Exactly as shown in RESEARCH.md: PA5 toggle every 500ms. Includes `stm32f4xx_hal.h`, declares `SystemClock_Config` and `MX_GPIO_Init` prototypes, main() calls HAL_Init, SystemClock_Config, MX_GPIO_Init, then infinite loop with TogglePin + Delay(500). MX_GPIO_Init enables GPIOA clock, configures PA5 as OUTPUT_PP. Include thorough educational comments explaining every HAL call.
    - `samples/knight-rider/main.c` -- PA5 through PA8 (4 LEDs) chase pattern. MX_GPIO_Init configures PA5, PA6, PA7, PA8 as OUTPUT_PP (either individually or as a combined pin mask). Main loop sweeps left then right: set each pin high, delay 100ms, set low, move to next. Educational comments explain the chase pattern and multi-pin GPIO.
    - `samples/button-led/main.c` -- PA0 configured as INPUT with PULLUP, PA5 as OUTPUT_PP. Main loop reads PA0, writes inverse to PA5 (button pressed = low = LED on). HAL_Delay(50) for debounce. Educational comments explain input reading, pull-up configuration, and debounce concept.
