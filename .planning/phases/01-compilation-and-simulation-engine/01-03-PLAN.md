<task type="auto">
  <name>Task 1: Create HTTP routes, WebSocket handler, and server entry point</name>
  <files>
    src/server/routes/compile.ts, src/server/routes/run.ts,
    src/server/routes/stop.ts, src/server/routes/samples.ts,
    src/server/ws/handler.ts, src/server/index.ts
  </files>
  <action>
    **Create route handlers (each is a small, focused module):**

    `src/server/routes/compile.ts`:
    - Export `async function handleCompile(req: Request): Promise<Response>`
    - Parse JSON body: `{ code: string }`
    - Validate: code must be non-empty string, return 400 if not
    - Call `compile(code)` from compiler module
    - Return `Response.json(result)` with status 200 on success, 200 on compilation failure (errors are in the response body, not HTTP errors)
    - Store compilationId -> binaryPath mapping in a module-level Map for use by the run route

    `src/server/routes/run.ts`:
    - Export `async function handleRun(req: Request, server: Server): Promise<Response>`
    - Parse JSON body: `{ compilationId: string, speed?: number, timeout?: number }`
    - Validate: compilationId must exist in the compilation map, return 404 if not
    - Call `startSimulation()` with onEvent callback that publishes to WebSocket topic `sim:{simulationId}` via `server.publish()`
    - onExit callback: publish `{ type: "sim_exit", data: { exitCode, signal } }` to the topic, then clean up compilation temp dir
    - Return `Response.json({ simulationId, wsUrl: "ws://localhost:${port}/ws?simulationId=${simulationId}" })`

    `src/server/routes/stop.ts`:
    - Export `async function handleStop(req: Request): Promise<Response>`
    - Parse JSON body: `{ simulationId: string }`
    - Call `stopSimulation(simulationId)`
    - Return `Response.json({ stopped: true })` or 404 if not found

    `src/server/routes/samples.ts`:
    - Export `function handleListSamples(): Response` -- reads samples/ directory, returns JSON array of `{ name, title, description }` for each sample. Title/description extracted from the first comment block in each main.c.
    - Export `async function handleGetSample(name: string): Promise<Response>` -- reads `samples/{name}/main.c`, returns `Response.json({ name, files: [{ name: "main.c", content: "..." }] })`. Return 404 if sample not found.

    **Create WebSocket handler:**

    `src/server/ws/handler.ts`:
    - Export a `wsHandlers` object compatible with `Bun.serve()` websocket config: `{ open, message, close }`
    - Track which WebSocket clients are subscribed to which simulation via a `Map<string, Set<ServerWebSocket>>` (simulationId -> clients)
    - `open(ws)`: Extract simulationId from `ws.data.simulationId`, subscribe to topic `sim:{simulationId}`, add to client tracking map
    - `message(ws, msg)`: Parse JSON. If `type === "gpio_input"`, this is for Phase 2 button support -- for now, log and ignore. Reserve for future use.
    - `close(ws)`: Unsubscribe from topic, remove from client tracking map. If this was the last client for a simulation, call `stopSimulation()` to clean up the orphaned process.

    **Create server entry point:**

    `src/server/index.ts`:
    - Use `Bun.serve()` with built-in `routes` object following RESEARCH.md Pattern 3 exactly
    - Register all routes: `"POST /api/compile"`, `"POST /api/run"`, `"POST /api/stop"`, `"GET /api/samples"`, `"GET /api/samples/:name"`
    - For `/api/run`, pass `server` instance to the handler
    - `fetch(req, server)`: Handle WebSocket upgrade for `/ws` path. Extract `simulationId` from query params. Call `server.upgrade(req, { data: { simulationId } })`.
    - `websocket`: Use handlers from ws/handler.ts
    - Port: Read from `process.env.PORT || 3000`
    - Log startup: `console.log("STM32 Virtual Test Bench server running on port ${port}")`
    - Handle SIGINT/SIGTERM: call `stopAllSimulations()` before exit

    **CORS headers:** Add `Access-Control-Allow-Origin: *` and `Access-Control-Allow-Methods: GET, POST, OPTIONS` to all responses. Handle OPTIONS preflight. This is needed for Phase 2 frontend.
