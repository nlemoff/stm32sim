<objective>
Fix SimConnection.disconnect() so it only closes the WebSocket without clearing event handlers. This ensures gpio_init, gpio_write, and sim_exit handlers registered once at application startup remain active across multiple simulation run/stop cycles.

Purpose: The verification report found that after the first Stop, the second Run opens a new WebSocket but has no handlers -- LEDs and pin table do not update. This is because disconnect() calls handlers.clear() and resets closeHandlers, destroying one-time registrations from index.ts and toolbar.ts.

Output: Modified websocket.ts where disconnect() closes the socket but preserves the handler dispatch table.
</objective>
