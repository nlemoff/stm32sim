<task type="auto">
  <name>Task 2: Create CodeMirror editor, API/WebSocket clients, state machine, and serve static files</name>
  <files>src/client/index.ts, src/client/editor/editor.ts, src/client/sim/api.ts, src/client/sim/websocket.ts, src/client/sim/state.ts, src/server/index.ts</files>
  <action>
    1. Create src/client/editor/editor.ts:
       - Export `initEditor(container: HTMLElement): EditorView` — creates CodeMirror 6 with `basicSetup`, `cpp()` language, `oneDark` theme, and custom theme extension for full-height (`& { height: 100% }`, `.cm-scroller { overflow: auto }`).
       - Export `getCode(): string` — returns `view.state.doc.toString()`.
       - Export `setCode(code: string): void` — dispatches a transaction replacing the full document. Uses `view.dispatch({ changes: { from: 0, to: view.state.doc.length, insert: code } })`.
       - Export `setupFileUpload(inputEl: HTMLInputElement): void` — listens for change event, reads first file via `file.text()`, calls `setCode()`. Resets input.value after. Accepts .c and .h files.
       - Default editor content: a simple STM32 template with `#include "stm32f4xx_hal.h"`, main function, HAL_Init, while(1) loop with HAL_Delay.

    2. Create src/client/sim/api.ts:
       - Export `compile(code: string)` — POST /api/compile, returns parsed JSON.
       - Export `run(compilationId: string, speed?: number)` — POST /api/run with { compilationId, speed }, returns parsed JSON.
       - Export `stop(simulationId: string)` — POST /api/stop, returns parsed JSON.
       - Export `listSamples()` — GET /api/samples, returns parsed JSON.
       - Export `getSample(name: string)` — GET /api/samples/:name, returns parsed JSON.
       - Use empty string for API_BASE (same origin). All functions are async, return the parsed response. Throw on non-ok responses with error message from body.

    3. Create src/client/sim/websocket.ts:
       - Export class `SimConnection`:
         - `connect(simulationId: string): void` — opens WebSocket to `ws://${location.host}/ws?simulationId=${simulationId}`. Sets up onmessage to parse JSON and dispatch to handlers. Sets up onclose to call registered close handlers.
         - `on(type: string, handler: (event) => void): void` — registers a handler for an event type. Support '*' for wildcard.
         - `sendGpioInput(port: string, pin: number, state: number): void` — sends JSON message `{ type: "gpio_input", port, pin, state }` if ws is open.
         - `disconnect(): void` — closes the WebSocket.
         - `onClose(handler: () => void): void` — register close handler.
         - Internal: store handlers in a Map<string, Function[]>. Clear handlers on disconnect.

    4. Create src/client/sim/state.ts:
       - Export type `SimStatus = 'idle' | 'compiling' | 'running' | 'stopped' | 'error'`.
       - Module-level `currentStatus: SimStatus = 'idle'` and `listeners: StatusListener[]`.
       - Export `getStatus(): SimStatus`.
       - Export `setStatus(status: SimStatus, detail?: string): void` — updates and notifies listeners.
       - Export `onStatusChange(fn: (status: SimStatus, detail?: string) => void): void`.

    5. Create src/client/index.ts — the application bootstrap:
       - Import and call `initEditor` on `#editor-container`.
       - Import `listSamples`, `getSample` from api.ts. On DOMContentLoaded: fetch samples list, populate `#sample-select` dropdown. On dropdown change, fetch sample code and call `setCode()`.
       - Import and call `setupFileUpload` on `#file-upload` input.
       - Export the editor view, SimConnection instance, and state functions so Plan 03 can import them.
       - NOTE: Do NOT wire up Run/Stop buttons yet — Plan 03 handles that. Just initialize the editor and sample loading.

    6. Modify src/server/index.ts to serve static files:
       - After all API route checks and before the 404 fallback, add static file serving.
       - Use `join(process.cwd(), 'dist')` as the static directory (import `join` from `path`).
       - Map pathname to file path: if pathname is '/', serve 'index.html'. Otherwise serve the requested path.
       - Use `Bun.file(filePath)` and check `await file.exists()`. If exists, return `new Response(file)`.
       - This goes AFTER all API routes and WebSocket upgrade, BEFORE the 404.
       - Wrap static file responses in `withCors()` too.

    7. Build and verify: `bun run build:client && bun test` (existing tests must still pass).
