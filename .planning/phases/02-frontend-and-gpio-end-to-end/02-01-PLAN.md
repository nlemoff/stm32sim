<objective>
Add stdin-based GPIO input injection to the simulation pipeline so that external input (virtual button presses from the frontend) can update GPIO IDR registers and be read by HAL_GPIO_ReadPin().

Purpose: GPIO-04 requires the frontend to send input signals to running firmware. The mechanism is: frontend -> WebSocket gpio_input message -> server writes JSON to subprocess stdin -> C runtime updates IDR -> HAL_GPIO_ReadPin returns new value. This plan implements the full backend path.

Output: Modified C HAL stubs with non-blocking stdin reader, modified server process-manager with stdin pipe, modified WS handler forwarding gpio_input messages, and integration test proving the round-trip.
</objective>
