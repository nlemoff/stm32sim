  <action>
    1. Create src/client/gpio/gpio-state.ts — the GPIO state model (NOT the DOM):
       - Export class `GpioState` with a `Map<string, PinState>` keyed by "port:pin".
       - Export interface `PinState { port: string; pin: number; direction: 'input' | 'output' | 'unknown'; mode: string; value: number; }`.
       - `handleGpioInit(data: { port: string; pins: number[]; mode: string }): void` — creates PinState entries. Direction derived from mode: if mode === 'input' then 'input', else 'output'. Initial value = 0.
       - `handleGpioWrite(data: { port: string; pin: number; state: number }): void` — updates value for the pin. If pin not yet in map, creates an entry with direction 'output'.
       - `getPin(port: string, pin: number): PinState | undefined`.
       - `getAllPins(): PinState[]` — sorted by port then pin number.
       - `reset(): void` — clears all entries.

    2. Create src/client/gpio/led-panel.ts:
       - Export `initLedPanel(container: HTMLElement): void` — stores reference to the container element.
       - Export `updateLed(port: string, pin: number, state: number): void` — finds or creates the LED element for port:pin. LED element structure: a wrapper div containing a circular div.led and a label span showing "Px#" (e.g., "PA5"). If the LED does not exist yet, create it and append to the container. Set class to `led led-on` if state=1, `led led-off` if state=0. Store LEDs in a Map<string, HTMLElement>.
       - Export `clearLeds(): void` — removes all LED elements from the container and clears the map.
       - Per Pitfall 2 from research: `clearLeds()` must be called when starting a new simulation.

    3. Create src/client/gpio/pin-table.ts:
       - Export `initPinTable(container: HTMLElement): void` — creates an HTML `<table>` with headers: Port, Pin, Direction, State. Appends to container.
       - Export `updatePinTable(pins: PinState[]): void` — replaces the table body with rows from the given pin array. State column shows "HIGH" (green text) or "LOW" (dim text). Direction column shows "IN" or "OUT" with appropriate styling.
       - Export `clearPinTable(): void` — empties the table body.

    4. Create src/client/gpio/button-panel.ts:
       - Export `initButtonPanel(container: HTMLElement, onPress: (port: string, pin: number, state: number) => void): void` — stores reference and the callback.
       - Export `addButton(port: string, pin: number): void` — creates a virtual button element for the given input pin. Button shows label "P{port}{pin}" (e.g., "PA0"). On mousedown: call onPress(port, pin, 1) (pressed). On mouseup AND mouseleave: call onPress(port, pin, 0) (released). This simulates holding a physical button. Use the `vbtn` CSS class. Store buttons in a Map to avoid duplicates.
       - Export `clearButtons(): void` — removes all button elements.
       - Per research recommendation: auto-detect input pins from `gpio_init` events with mode "input" and call `addButton()` automatically. This is handled in the wiring code (index.ts), not here.

    5. Verify build: `bun build src/client/index.html --outdir=dist`
